---

## 1. 项目概述

**OneCup** 是一个面向印染厂的新面料开发管理系统。核心业务是面料的开发流程管理:新面料立项后,经历织造、流程、染色等工艺环节的录入与沉淀,开发完成后归档为产品,进入产品管理。归档内容包含面料基本信息以及开发过程中的织造工艺、流程工艺、染色工艺数据。

**部署形态:** 企业内部系统,单体应用,用户规模为几十到几百人。

## 2. 技术选型

| 层面 | 选型 | 说明 |
|------|------|------|
| **后端框架** | .NET 10 (LTS) | 跨平台,企业级,支持到 2028 年 |
| **数据库** | PostgreSQL | 强关系型场景,多表关联、状态流转、高一致性 |
| **ORM** | EF Core (Code-First) | LINQ 强类型查询,自动迁移 |
| **前端框架** | React 18 + TypeScript | 类型安全,前后端数据契约清晰 |
| **前端脚手架** | Arco Design Pro (Vite 版) | 开箱即用的中后台方案,自带布局/权限路由/请求封装 |
| **构建工具** | Vite | 启动快,热更新快 |
| **UI 组件库** | Arco Design React | 字节出品的企业级组件库 |
| **前端状态管理** | Redux (全局 UI 状态,Arco Pro 自带) + React Query (服务端数据) | 全局 UI 状态走 Redux;业务数据缓存走 React Query |
| **认证** | JWT Token | 无状态,前后端分离标准方案 |
| **部署** | Linux + Docker | 轻量,易维护,CI/CD 友好 |

### 3.3 后端分层依赖方向(严格单向)

```
Api → Application → Domain
Api → Infrastructure → Application → Domain
Domain 不依赖任何人(零依赖,纯 C#)
```

- **Domain**: 实体、值对象、领域规则。只引用 .NET BCL,不引用 EF Core。可脱离数据库单元测试。
- **Application**: 业务用例、DTO、服务接口(`IRepository`、`IUnitOfWork`)。依赖 Domain。
- **Infrastructure**: EF Core 实现、数据库访问、外部服务。依赖 Application(实现其定义的接口)。
- **Api**: 控制器、中间件、Program.cs。依赖 Application + Infrastructure。

## 4. 业务模块划分

业务域按依赖层次分为四层:下层永远不被上层感知(单向引用)。

### 4.1 模块清单

| # | 模块 | 层级 | 核心对象 | 提供的功能(概要) |
|---|------|------|----------|-------------------|
| 1 | **基础设置** | 基础数据 | 编号规则、单位 | 各类单据/物料的自动编号规则配置;计量单位的维护与换算 |
| 2 | **设备管理** | 主数据 | 设备、设备参数、工艺模式 | 设备台账;设备技术参数;同一设备的不同工艺模式预设(如烧毛机 5 种烧法) |
| 3 | **原料物料** | 主数据 | 纱线、坏布、染料、化学品 | 各类物料档案的增删改查,供工艺录入时引用 |
| 4 | **客户管理** | 主数据 | 客户 | 客户档案维护 |
| 5 | **颜色对色** | 主数据 | 颜色库、颜色标准、对色记录 | 颜色档案与标准维护;来样对色记录 |
| 6 | **样品文档** | 主数据 | 样品图片、工艺单文件 | 样品/单据文件的上传、存储与查看 |
| 7 | **面料开发** | 核心流程 | 面料开发单 | 立项、工艺录入入口、状态流转、归档触发 |
| 8 | **织造工艺** | 核心流程 | 织造工艺单 | 录入织造相关工艺数据,挂载于面料开发单 |
| 9 | **流程工艺** | 核心流程 | 流程工艺单 | 录入印染流程工序数据,挂载于面料开发单 |
| 10 | **染色工艺** | 核心流程 | 染色工艺单 | 录入染色配方与参数,挂载于面料开发单 |
| 11 | **产品管理** | 核心流程 | 成品面料 | 归档成品的检索、查看、统计 |
| 12 | **系统管理** | 支撑 | 用户、角色、权限 | 用户账号、角色、权限分配 |

### 4.2 分层依赖关系

```
【第一层:基础数据】(系统级配置,几乎纯字典表)
   1. 基础设置 (编号规则 / 单位)
        ↑ 被全局引用

【第二层:主数据】(业务资源档案,被上层流程引用)
   2. 设备管理     (设备 / 参数 / 工艺模式)
   3. 原料物料     (纱线 / 坏布 / 染料 / 化学品)
   4. 客户管理
   5. 颜色对色     (颜色库 / 标准 / 对色记录)
   6. 样品文档
        ↑ 被工艺录入引用

【第三层:核心流程】(主线,有状态流转,引用主数据)
   7. 面料开发 (主轴)
       ├── 8. 织造工艺  (并列, 挂 FabricDevelopmentId)
       ├── 9. 流程工艺  (并列, 挂 FabricDevelopmentId)
       └── 10. 染色工艺 (并列, 挂 FabricDevelopmentId)
   11. 产品管理 (归档后的成品,只读检索)

【支撑层】
   12. 系统管理 (用户 / 角色 / 权限)
```

### 4.3 核心业务流转

> 面料开发单是主轴,织造/流程/染色三种工艺**并列挂载**其下,互不依赖。
>
> **归档是关键流转点:** 面料开发单状态变为"已归档"时,面料信息 + 三种工艺数据被锁定(变只读),同时生成一条产品记录进入产品管理模块。
